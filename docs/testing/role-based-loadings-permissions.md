# QA Checklist: Role-Based Loadings Permissions

**Feature:** Расширенная видимость отдела + per-role edit gating на вкладках Разделы/Отделы (`/tasks`)
**Spec:** [docs/superpowers/specs/2026-04-28-role-based-loadings-permissions-design.md](../superpowers/specs/2026-04-28-role-based-loadings-permissions-design.md)
**Date:** 2026-04-28

---

## Подготовка тестовых аккаунтов

Нужны 6 аккаунтов (или 1 admin для переключения ролей):

| Роль | Скоп | Тестируется |
|---|---|---|
| `admin` | Всё | Доступы без ограничений + bulk shift на любой отдел |
| `subdivision_head` | Своё подразделение | Видимость, edit без cross-dept ограничений |
| `department_head` | Свой отдел | Полные права в отделе + bulk shift |
| `team_lead` | Своя команда | Видит весь отдел, edit только команды + own |
| `project_manager` | Управляемые проекты | Edit на своих проектах |
| `user` | Своя команда | Видит весь отдел, edit только own |

Перед тестированием убедись что юзеры реально привязаны к команде/отделу/подразделению (через `view_users`).

## 2. Регрессионные сценарии

### RG1. Resource Graph (`/planning`)

- [ ] Видит только свою команду (как было раньше)
- [ ] Drag-to-resize работает на своих загрузках
- [ ] Никаких изменений в UI

### RG2. Kanban на `/tasks`
- [ ] Открыть Kanban tab
- [ ] Карточки видны как раньше
- [ ] Создание/редактирование задач работает

### RG3. Модалки `/dashboard/dev/tasks`
- [ ] Открыть `/dashboard/dev/tasks`
- [ ] Все модалки открываются нормально
- [ ] Никаких regression

### RG4. RLS политики на `loadings`
- [ ] Прямой запрос `select * from loadings` через Supabase REST под юзером
- [ ] Старая политика работает: возвращает только разрешённые

### RG5. BulkShift для admin
- [ ] Войти как admin → /tasks → Departments
- [ ] Открыть BulkShift на любом отделе
- [ ] Работает без ошибок

### RG6. Restricted projects
- [ ] Создать restricted project
- [ ] Войти как user (не админ)
- [ ] На вкладках разделов/отделов не видит загрузки на restricted-проекте
- [ ] Попытка драг — server отвечает "Загрузка не найдена"

### RG7. Capacity edit
- [ ] Войти с правом `sections.capacity.edit`
- [ ] Открыть Sections → клик по ёмкости раздела
- [ ] Редактирование работает (без изменений)

## 3. Edge cases

### EC1. Юзер без отдела
- [ ] У юзера `profile.department_id = null`
- [ ] Открывает /tasks → видит пустой список (мандаторный фильтр блокирует)
- [ ] Никаких ошибок в консоли

### EC2. Юзер с двумя ролями (user + team_lead)
- [ ] В user_roles две записи
- [ ] Edit правила объединяются (видит drag-handles на своих + командных загрузках)

### EC3. Загрузка без team/dept у responsible (legacy)
- [ ] У какой-то загрузки `loading_responsible.team_id = null`
- [ ] User видит её через `.own` если он сам responsible
- [ ] Team_lead не видит drag-handles (без teamId match)

### EC4. Создание загрузки для себя в чужом отделе (невозможно)
- [ ] Открыть модалку создания
- [ ] В списке сотрудников нет юзеров из чужих отделов (для restricted ролей)

### EC5. Stale cache 10 минут
- [ ] Залогинены, открыта /tasks
- [ ] Админ через 5 минут добавил/убрал permission у юзера
- [ ] Юзер должен увидеть изменение либо через `useFilterContext` invalidation, либо после reload

### EC6. Двойной клик на drag-handle
- [ ] Не задваивает мутацию (existing optimistic update protection)

### EC7. Concurrent reassignment
- [ ] Юзер открыл модалку редактирования загрузки
- [ ] Параллельно админ переназначил эту загрузку в другой отдел
- [ ] Юзер нажимает Сохранить → server отвечает "Нет прав на редактирование загрузки" → toast error

## 4. Smoke tests (pre-deploy)

```
✅ STAGE 1: User
1. Войти как user → /tasks → Sections
   - [ ] Виден весь свой отдел (несколько команд)
   - [ ] Drag-handles только на своих загрузках
   - [ ] Клик на чужой загрузке → открывается read-only модалка
   - [ ] Кнопка "Создать" видна только на своей строке

✅ STAGE 2: Team lead
2. Войти как team_lead → /tasks → Departments
   - [ ] Drag-handles на загрузках своей команды
   - [ ] Drag-handles НЕТ на чужих командах в своём отделе
   - [ ] Кнопка Bulk Shift НЕ видна

✅ STAGE 3: Department head
3. Войти как department_head → /tasks → Departments
   - [ ] Drag-handles на всех загрузках своего отдела
   - [ ] Кнопка Bulk Shift видна, работает
   - [ ] Drag-handles нет на чужих отделах (если каким-то образом видны)

✅ STAGE 4: Admin
4. Войти как admin
   - [ ] Existing functionality intact
   - [ ] BulkShift работает на любом отделе
```

## 5. Sentry monitoring

После деплоя в продакшен следить за:

- `[SECURITY] applyMandatoryFilters` — попытки байпаса фильтров (должно быть 0)
- `canEditLoading: denied` — нормальные отказы (часто ожидается, не алерт)
- `canEditLoading: denied` с одинаковым `userId` массово в течение часа → подозрение на байпас → разобраться
